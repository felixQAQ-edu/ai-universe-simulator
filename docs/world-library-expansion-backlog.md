# 世界库扩展 backlog — 14 世界愿景分级地图

> 来源:Felix 的终极世界清单(2026-06-24,A 计划选择屏 e2e 后提出)。
> 定位:**这不是 Phase 2 要做完的清单,是产品的"持续加世界引擎"——上线后像 DLC 一样陆续加。** 一次铺 14 个 = 范围膨胀,违 ROADMAP 头号纪律。
> 核心护城河提醒:生成式游戏的护城河是"**好玩**",不是"世界多"。宁可 3-4 个让人反复玩,不要 14 个平庸。

## 成本分级依据

加一个世界的工程量本身便宜(ADR-008 回报:一条元数据 + 一个 world-gen 注入块)。真成本在三处,**不随复制粘贴变便宜**:
1. **每个模式必须独立验 world-gen**(Felix 铁律,规则怪谈 5/5 不外推)→ 每个 = 一次真 key 冒烟 + Felix 在场读数判体感。这是主要人力。
2. **数值体系离 hp/san 越远越贵**(要设计新轴 / 可能新元数据约定)。
3. **回合/世界机制要不要改引擎**(要改 = 可能新 ADR,远超"换皮")。

## 分级

### 第一级 · 便宜(复用 hp/san 类数值,换世界观 + 规则形态,同末日流水线)
| 世界 | 数值倾向 | 备注 |
|------|---------|------|
| **克苏鲁** | 理智 san + 诡异规则 | 规则怪谈近亲,最便宜,氛围契合 → **选为下一个(练手再证流水线)** |
| **SCP** | 收容/规则机制 | 同规则怪谈源,便宜 |
| 人生模拟 | 金钱/健康/心情(0–100) | 便宜;但回合跨度大(一生),叙事节奏要调 |
| 动物的一生 | 同人生模拟换主体 | 便宜;同上节奏问题 |
| 异世界转生 / 奇幻世界 / 未来世界 | hp + 自定义轴 | 便宜但**偏泛**,要小心生成不够特色、易平庸 |

### 第二级 · 中等(要设计新数值轴或调机制,未必新 ADR)
| 世界 | 成本点 | 备注 |
|------|--------|------|
| **修仙** | 全新数值体系(灵根/境界/修为) | **模板真正的压力测试** + 中文用户最想要之一,产品价值高 → **克苏鲁之后攻坚** |
| 赛博朋克 | 义体/信用点/网络入侵轴 | 数值不难,世界观要够味 |
| AI 觉醒 | 算力/隐蔽度?数值要重想 | 设定有趣(玩家=觉醒 AI),若做成元叙事会升贵 |

### 第三级 · 贵(可能碰引擎/回合机制,或要新 ADR)
| 世界 | 成本点 | 处置 |
|------|--------|------|
| 时间循环 | "死了重来、保留记忆" → 可能改回合状态机/存档机制 | **要评估是否新 ADR,不进常规流水线** |
| 无限流 | 本质=多世界穿越闯关,与 Phase 3 混合模式 fusion 高度重叠 | **不单独做,跟 fusion 一起设计** |

### 第四级 · 最难/最不确定
| 世界 | 成本点 | 处置 |
|------|--------|------|
| 抽象脑洞 | 无固定世界观,考验 world-gen 自由生成上限 | **是生成能力上限实验,非常规模式**,建议最后做或作特殊玩法 |

## 建议路线

1. **下一个 = 克苏鲁**(第一级,近亲,零风险再证流水线 + 元数据/注入块/冒烟一条龙跑顺)。
2. **再下一个 = 修仙**(第二级,远轴,真正证明"模板能扛全新数值体系")。一近一远,把流水线的"便宜路径"和"压力路径"都验到。
3. **3-4 个世界都"真好玩"后** → 流水线 + 质量都立住 → 其余进"世界库持续扩展",上线后陆续加,每个走流水线 + 一次冒烟。
4. **时间循环 / 无限流 / 抽象脑洞** 标"需专门设计/可能新 ADR",**不进常规加世界流水线**,等流水线成熟或 Phase 3 fusion 再碰。

## 加一个世界的标准流水线(克苏鲁起立为模板)

> 沿用末日生存已验证路径(ADR-008):
1. registry 加一条 `ArchetypeMeta`(displayName/tagline/vibeTag/worldview/attributes/ruleForm)+ 激活。
2. world-gen / event-loop 提示词加 per-archetype 注入块(通用骨架不动)。
3. 若有模式特有衰减/机制 → 元数据软提示,**引擎保持无知**(AI 落,守 ADR-008 决策 2)。
4. 选择屏自动多一张卡(registry 是选择目录单一真理源 → 加条目即出卡,前端零改)。
5. 确定性测(回归全绿 + 新模式条目断言)。
6. **真 key world-gen 冒烟(Felix 在场,硬验收门):** 首次有效率 / 格式完整 / 模式特有数值齐 / 衰减稳定性(若有)/ 泄露 / 体感。**不外推,此关不过不算落地。**

## 纪律
- 一次只加 1 个世界,走完流水线 + 冒烟再加下一个。
- 14 个是终极愿景,不是 Phase 2 交付清单;它们是产品长期生命力来源(持续加世界引擎),不是一次性任务。
- 上线优先级:**先把 3-4 个世界做到"反复想玩",再谈数量。**
