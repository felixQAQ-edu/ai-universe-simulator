## 2. 本轮分档顺序(每条带退出条件)

### 现在就做(真打磨 · 便宜 · 马上见效)

**#3 数值行为化**
- 做法:每个数值轴定义 3 档「阈值 → 行为/叙事」描述(如 气血 >50 可连续战斗 / <30 行动迟缓 / <10 吐血),挂在现有 per-archetype 轴元数据(ADR-009 `axisRole`);AI 读元数据染叙事 + 前端显示当前所在档。
- 退出条件:每轴 3 档描述 + 前端档位显示,做完即止。
- 待定子决策:阈值档位是**只染叙事**(便宜、真打磨)还是**真开关选项**(滑向机制层、与 #4 重叠)——开 #3 时定。

**#8 静态氛围图**(见 §3 第 1 档)
- 做法:每世界 1 张人工审过的静态底图,**只画场景/ambient、不画人物或具体事件**,正文背后做氛围背景。
- 退出条件:每世界 1 张图 + 可读性处理(压暗/降饱和/底部遮罩),纯前端零后端,做完即止——**不漂向动态生成**。

### 近期(需一条小 ADR)

**#1 选择反馈(定性版)** —— **已落地(ADR-011,2026-07-01)**
- 做法:AI 给每个选项标一句「风险/代价提示」当**叙事元数据**(引擎只读不裁决,守 ADR-008);前端在选项上呈现。
- **先不做真概率掷骰**:「成功率 80%」若背后无真掷骰、真扣状态,玩多了露馅、反而更假。先上定性张力(「天劫已锁定你」「损道基」)。
- 勘察订正:`hint` 字段/wire/校验/渲染**端到端已存在**、引擎纯 `deepCopy` 透传,唯一缺的是**提示词引导**——故本轮**无需动 schema/引擎/`schemaVersion`**(比原估「选项元数据 schema 扩展」更轻)。ADR-011=**范围/边界 ADR**(立字「hint=定性元数据、引擎只读永不 gate/掷骰」),纯提示词(world-gen + event-loop lockstep)+ 前端样式 + 文档。真概率机制拆为 **#1.5** 冻结(见下)。

### 冻结 / 不动

**#1.5 选项真概率掷骰**(从 #1 拆出,冻结) —— 交叉链 [ADR-011](adr/ADR-011-action-hint-narrative-metadata.md)
- **构想**:选项不止一句定性 hint,而是 **AI 定概率 + 引擎掷骰 + 状态结算**——玩家选项有真随机、真后果(选它可能成/败,引擎据概率掷、把结果落进 state 影响后续),把「选择有重量」从定性推到定量博弈。
- **为何冻结(与 ADR-011 边界互补)**:① **违 ADR-008 引擎无知**——引擎要认识「概率」并据此裁决结果,从「只读透传」变成「按 hint 掷骰、决定叙事走向」,是承重架构改动(schema + 引擎 + ADR);② **依赖状态层**(掷骰结果进 state、影响结算),Phase 2 打磨期尚无此接缝;③ 精确百分比一旦显示,玩家易据数字反推最优解、把体验从「读氛围抉择」拉成「刷概率」,反噬沉浸。
- **性质**:#1 定性版的自然后续档,独立大工作单元;等**状态层(账号/云存档/持久世界记忆)就位**后再议,届时另起 schema + 引擎 + 独立 ADR。**郑重入档,勿丢。**

**主题 A · 持久世界记忆** —— 见 §4。本轮不起任何代码、不起 ADR。

## 3. #8 氛围图:四档静动边界

| 档 | 形态 | 运行时成本 | 边界 |
|----|------|-----------|------|
| 第 0 档 | 纯 CSS 卡片氛围(抽象质感) | 零 | **已完成** |
| **第 1 档** | 每世界一张静态底图 | 近零(对象存储 + CDN) | **本轮做** |
| 第 1.5 档 | 每世界一小套 + AI 选图(`sceneTag` 枚举,前端按 tag 换图) | 近零(机制与 `discoveredRuleIds` 同形:AI 标枚举、引擎透传、前端映射、缺失回退) | 可选加档,**不与第 1 档同批** |
| 第 2 档 | 运行时 AI 实时生成 | 高 | **冻结**(另一个项目,见下) |

**第 2 档为何是另一个项目(四条硬成本线)**:
1. 需单独接文生图 provider(国产,延迟/合规),且其 API 形状现有 OpenAI 兼容 chat 抽象接不住;
2. 单图价 ≈ 文字回合的几十~上百倍,砸穿成本模型;
3. 延迟 5–15s,砸掉 ~1s TTFT 手感;
4. 图像审核法定且独立于文字审核 → 逼解冻 ADR-004 并扩展到图像;另有风格漂移/一致性坑。

**关键洞察**:第 1 / 1.5 档安全,因为图是**有限的、一次性人工审过的集**——合规、一致性、成本三难一起消失。所有麻烦都长在「运行时生成」这一步上;面向中国产品,这是死守静态侧的最强理由。

## 4. 岔路结论:混合模式 vs 个人史(2026-06-30)

**背景**:十条反馈实质在提一个与原定杀手锏(混合模式)不同的杀手锏——「AI 记住玩家的一生 / 个人史」(主题 A)。两者都强,但单人 10–20h/周近期只能深做一个。

**考量**:个人史含金量极高,但地基压在尚未做的账号 + 云存档,抬升单回合成本、撞日志摘要压缩,是多 ADR、多周的下个大版本。

**拍板**:**押混合模式**(与 ROADMAP §七「旗舰模式 = 最大差异点」既有定位一致)。个人史作为**有意识权衡后的 runner-up**入档冻结——非遗忘,是择序。

**后续**:混合模式启动时另起 ADR(架构 + 概念融合自洽 + 彩蛋验证),非本轮;个人史若未来重启,先补账号+云存档基础设施。

## 6. 独立小单元 · 末日「饥饿」轴改名(legibility,从 #3 拆出)

**问题**:末日 `hunger` 轴是 depletion 轴(高值=温饱、`≤0`=饿死),但 displayName 叫「**饥饿**」——高值=不饿,语义与值方向反直觉(面板「饥饿 80」其实是吃饱)。理想改成「温饱」/「满足度」(高=好,与值方向自洽)。

**为何拆出(#3 数值行为化 Slice A enumerate 时验)**:「饥饿」一词不止 registry 显示名,已**织进 prompt / 规则 / 事件文案**,改 displayName 不连带改这些会造成「面板名 vs 叙事名」两套叫法。引用面:
- `prompts/world-gen.md`(示例注入 `hunger(饥饿,…)`、condition 示例「饥饿而亡」)
- `prompts/event-loop.md`(AXIS_IMAGERY:`末日 hunger→饥饿/腹中空乏/虚脱无力`)
- `server/.../eventloop/TurnPromptBuilder.java`(运行时 AXIS_IMAGERY 副本 `"hunger" → "饥饿/腹中空乏/虚脱无力…"`)
- `server/.../worldgen/WorldGenPromptBuilder.java`(condition 示例「饥饿而亡」)
- registry `worldview`/`tagline`/`behaviorHint` 散文(「废土求生,饥饿是另一个敌人」「饥饿值随回合自然衰减」)
- `docs/CONTEXT.md` §三.14 元数据示例 + 多处 `{hp,hunger}` 叙述

**范围**:真正改名要一并改上述 prompt/imagery/condition 措辞(否则 UI↔叙事错位 / 一名两叫),属 prompt 改动,有触碰 A-1 叙事约束与 world-gen 行为的回归面 → **不在 #3 本单元做**,留作独立小单元(改名 + lockstep `.md`/运行时副本 + 真 key 冒烟)。

**本轮(#3)结论**:displayName 保留「饥饿」;band label(饱足 / 饥肠辘辘 / 濒临饿毙)与值方向(高=温饱)自洽,**不依赖改名**也读得通。

## 7. #8 衍生 · 世界记忆可视化(冻结)

> 来源:#8 顶部氛围图接入面只读勘察(2026-06-30)。勘察确认每回合 turn 契约只返回 `turn/status/数值/已发现规则/可选动作`,标题/危险度/基调全是 init 一次性生成、整局静态,无时间/天气字段。由此引出三条「让封面随世界演变」的候选——**三条共性:均依赖已冻结的主题 A(持久世界记忆,见 §1/§4),非 #8 纯前端静态底图范围,随主题 A / 混合模式大版本之后再议**。郑重入档,勿丢。

**① timeline 下发(最近一块踏板,成本最低)**
- **现状**:AI 每回合在 `stateUpdate` 产出 `state.timeline`(一句话世界线摘要,如「大比将启,杀机暗伏」),引擎已存,但 `buildDelta` **未转发**给客户端。
- **候选**:`buildDelta` 加一字段下发 `timeline`,前端在顶部标题带下显示「随回合变的一行场景摘要」。
- **性质**:动后端 turn 契约(`TurnDelta` + `buildDelta`),**非纯前端**;是「封面随情节演变」的最小可行第一步,三条里成本最低。

**② 换地点封面(establishing shot 随地点更新)**
- **构想**:进入新地点时顶部图随之更换(便利店→地下仓库→停车场),给「翻开新章节」感。
- **依赖**:① AI 判定「换地点」并返回结构化地点标识;② 地点→图映射;③ 每世界**多张**地点图(非一张)。等于把 #8 从「每世界一张」扩成「每地点一张 + 地点状态机」,触 [§3 第 1.5 档](#3-8-氛围图四档静动边界)以上、近内容管线。
- **性质**:依赖主题 A 的地点状态;入冻结。

**③ 细节随回合演变(同场景渐变:灯坏→货架空→玻璃裂→尽头多人)**
- **构想**:同一场景图随世界状态推进显现损坏/异变,体现「世界真的在变」。
- **依赖**:要么逐回合生成图(= 已冻结的 [§3 第 2 档](#3-8-氛围图四档静动边界),跨成本/延迟/图像审核四条硬线),要么预制「同场景多损坏度」图集 + 触发逻辑。本质是世界记忆的可视化,三条里最贵。
- **性质**:主题 A 核心果实之一,愿景里最惊艳但最贵;郑重入冻结,勿丢。

## 8. 融合模式 round 2 · 五对候选组合(发散定稿,冻结待选)

> 来源:混合模式 round 1(修仙×规则怪谈「识海遗蜕」,ADR-012/013)端到端验通(2026-07-08)后的 Project 窗口发散定稿。四个已激活世界两两组合剩五对,逐对压缩落档(一句定位 + 主推容器 + 独占爽点 + 风险一句),**冻结待选、不排期**;加一对融合 = ADR-013 已验证路径(combo 登记 + 换皮 + 融合 meta-prompt + 结局/资源经济文案,event-loop 自动走 `fusedAxes`,见 FINDINGS F-016 复用注)。

**① 规则怪谈 × 末日「守则即补给」——缺页的人防工程(★已选为 round 1.5 下一对,待勘察 + ADR)**
- **定位**:末日成因=守则失守;缺的页=人消失的原因;守则残页是硬通货。
- **爽点**:违规成生存刚需(饥饿轴顶着守则,守规矩会饿死、破规矩会死于别的)、真假裁决换成**物证与尸体**(不靠试探,靠废墟里的痕迹反推真伪)。
- **风险**:两套世界观自动咬合(都是「环境规训人」),难在别做平庸。

**② 规则怪谈 × 克苏鲁「规则是它的器官」——四百年藏书楼**
- **定位**:族规条数在各抄本里不一样;真族规是锁,锁着的东西在楼里。
- **爽点**:读规则涨禁忌知识、拖理智(读得越懂越疯);守规=喂它 / 违规=饿它(规则是活物的进食方式)。
- **风险**:近亲通婚——懒做就=「更吓人的规则怪谈」;活路=把「规则与神的关系」当主角。

**③ 末日 × 克苏鲁「为一盒胰岛素潜入神域」——深潜打捞队**
- **定位**:神没毁灭人类,只是没注意到;物资全在被神梦浸泡的深域,上浮须按减压舱仪式「正确地遗忘」。
- **爽点**:理智上配给表(san 是消耗性补给,和罐头一样要省着用)、拉莱耶捡罐头(在不可名状里做最世俗的事)。
- **风险**:必须**双区结构**(安全区/深域)撑两套逻辑,否则塌成「有触手的末日」。

**④ 末日 × 修仙「变强会招雷」——劫波之地**
- **定位**:失控天劫仍在游猎,修为越阈值雷落谁头;藏拙 vs 必须变强才能走出去。
- **爽点**:成长动词被判刑(变强=风险行为,境界轴从纯爽点变成双刃)、修为是不可再生能源(灵气死地,用一分少一分)。
- **风险**:难在别做平庸(设定一句话讲完,深度全靠守则与结局池设计)。

**⑤ 克苏鲁 × 修仙「飞升是捕捞」——方飞升的宗门(潜力天花板最高,留作压轴)**
- **定位**:飞升不是超脱,是被捕捞——天道是渔网,天劫=排酸、气运=育肥、飞升台=出栏口。
- **爽点**:对整个修仙 trope 栈做恐怖翻转(每个熟悉概念都换了恐怖含义)、修仙常识=调查工具(懂行才能看出哪里不对)、护道式赢法(局部赢、宇宙不赢,round 1 护道结局的精神续作)。
- **风险**:强弱平衡 + 与《道诡异仙》差异化(差异点=调查动词 + 守则玩法)。

**横向资产三条**:
- **容器模板预警**:「住进逝去强者的残余」已验证可复用(识海遗蜕),但**同批别连用两次以上**(第二次开始像套路)。
- **暗线活口**:识海遗蜕的低语 / 藏书楼那本书 / 天道渔网,可设计为**同一信号的三次显形**——schema 本支持 2–3 元融合(`archetypes` 2–3),远期可接三元;**现在只在写设定时留活口,不做任何实现**。
- **排序**:潜力 ⑤ > ① > ④ > ③ > ②;自洽难度 ② > ③ ≈ ⑤ > ④ > ①。①潜力第二、难度最低 → round 1.5 首选;⑤压轴留到管线更熟。

## 10. 静默 no-op 回合的玩家体感(独立小单元,round 1.5 冒烟发现)

- **现象**(F-017 取证顺带,2026-07-14):模型某回合输出无哨兵/空叙事 → 引擎按规格走**保守 no-op 降级**(turn++、数值冻结、复用上一组动作,后端响亮告警)——机制正确,但**玩家端无感**:叙事几乎空、数值不动、同一组选项再来一遍,体感=「我选了但游戏没反应」(人防第 3 局 T5 实测,恰逢补给紧张,放大了「领取不加」的挫败感)。
- **候选修法**(做时定):前端识别 no-op delta(数值全同 + 动作复用)给一句轻量提示(「时间悄然流过,什么都没发生……再试一次」之类的氛围话术)/ 或后端 no-op 时下发一句兜底叙事。均不动引擎结算语义。
- **状态**:入档待排期(体验补丁级,非主线);做时守 ADR-003 边界(纯展示或 no-op 路径文案,逻辑层零改)。

## 11. 人防工程「没有一夜一夜生存的感觉」(体感反馈,ADR-016 批顺手入档,2026-07-22)

- **现象**(Felix 线上真机整局体感,ADR-015 Slice 3 冒烟 ⑩):「守则即补给」组合玩下来**缺少「一夜一夜熬过去」的节奏感**——多日昼夜结构(E' 刀)在提示词里立了,但长局里昼夜切换的体感被叙事漂移冲淡,日子过着过着就糊成一片。对照:**识海遗蜕(单夜封闭)体感良好**——它的时间结构天然不需要「过日子」的节奏,侧面印证问题出在「多日循环」这种时间尺度更依赖持续的结构化提醒。
- **定性**:两股已知力量的合流,非新 bug——
  1. **F-017 平衡三刀的遗留张力**:E'/E'' 双刀刻意选了宽松方向(衰减挂钩行为、断粮倒计时),经济死局根治了,但「时间压力」也随之变软,昼夜钟摆的存在感下降;
  2. **CONTEXT §三.16 约定 (7) 的实例**:时间尺度是 per-combo 隐性设计参数,目前**只靠提示词软引导**——长局(几十回合)里模型对「现在是第几天/几点」的追踪会漂移,结构性时间感没有 schema/状态层的锚。
- **候选方向(做时定,记两条)**:
  1. **combo2 `extraDirectives` 昼夜钟摆强化**(提示词层,最轻):在现有附加指令槽里加「每回合必须报时段 + 昼夜切换须有仪式感叙事(熄灯/开闸/配给钟)」一类硬提醒;零 schema 改动,但仍是软引导,治标。
  2. **accumulation 轴「天数」使时间结构化可见**(状态层,较重):给 combo2 加一条 `axisRole=accumulation` 的「天数」轴(引擎天然支持,ADR-009;`lethal=false` 不触底)——时间从叙事装饰变成**状态真理之源里的数字**,AI 每回合落绝对值、前端面板可见「第 N 天」、bands 可配「第一夜/熬过三天/第七天」档;时间漂移被数值锚死。代价:动 combo2 轴集与 world-gen/event-loop 注入,需小设计稿。
- **状态**:照旧「想法入档,代码不动」,**不阻塞软启动**(体验打磨级,非闸门/合规级)。开刀条件=软启动真实玩家对 combo2 的同类反馈再现,或 round 2 新组合需要「多日循环」模板时一并做(届时候选 2 顺手成为所有多日组合的公共资产)。
- **交叉(2026-07-23)**:与 [`future-experience-backlog.md` ④「昼夜推进改变环境」](future-experience-backlog.md)是**同一缺口的两面**——本条是玩法体感缺时间锚,那条是视觉层缺时间锚,根因同为 CONTEXT §三.16 约定 (7) 只靠提示词软引导。**候选 2(「天数」accumulation 轴)一旦做,顺带就是视觉侧的依赖** → 两处一起做,别各做一半。
