## 0. 核心定位:禁忌知识的代价

克苏鲁**不换皮规则怪谈**。核心张力 = **"知道得越多越强、也越疯"**:玩家在"求真相/求力量"与"保住理智"之间挣扎。机制张力做进数值,不是文案口号。

## 2. 克苏鲁 registry 条目 + 签名轴

**数值轴**:`hp`(体力/伤势,复用)+ `san`(理智,复用)+ **`knowledge`(禁忌知识/真相)** 克苏鲁特有。

`knowledge` 设计(**累积型双刃**,跟末日饥饿"衰减型"相反——正好验流水线能否容纳不同行为的轴):
- 玩家主动求知/探索/接触禁忌之物 → `knowledge` **上涨**(AI 落,绝对值)。
- **双刃**:`knowledge` 高 → 解锁更强行动 / 看穿真相(力量);但 `knowledge` 越高 → **`san` 流失越快**(代价)。
- 这把"禁忌知识的代价"做成真机制,不是口号。

**元数据**(`ArchetypeMeta`,沿用末日字段形态):
- `displayName`:克苏鲁
- `tagline`(玩家可见钩子):如「凝视深渊,深渊回以低语。知道得越多,离疯狂越近。」
- `vibeTag`:如「深渊 · 疯狂」
- `worldview`:不可名状的古old恐怖、旧日支配者、禁忌典籍、海边小镇/古宅/学院等克苏鲁母题(喂注入块)
- `attributes`:hp(体力)/ san(理智)/ knowledge(禁忌知识),各带 displayName + 范围 0–100 + 行为提示(knowledge=累积双刃,见 §3)
- `ruleForm`:见 §3(禁忌知识渐揭,非真假守则)

## 3. 提示词注入块(world-gen + event-loop)

**通用骨架不动**(消毒/id/json_object/schema 固定);注入克苏鲁变量:

- **世界观**:克苏鲁母题(不可名状、旧日支配者、禁忌知识、理智崩坏)。
- **knowledge 轴行为(关键,AI 落语义)**:
  - 提示 AI:玩家主动求知/探索禁忌 → 回合 `stateUpdate` 让 `knowledge` 上涨(累积)。
  - **联动提示(本批新语义)**:`knowledge` 越高,本回合 `san` 流失应越快——让 AI 在 `stateUpdate` 里体现这个联动。**引擎不认识这个联动**(守 ADR-008 决策 1/2:引擎无知,AI 落);引擎照旧通吃 clamp/跳变/落账。
  - **绝不把"knowledge 影响 san"接进引擎**——破 key-agnostic 即违 ADR-008,是末日那个架构信号的坑。
- **规则形态(克苏鲁味,区别规则怪谈)**:不是"一纸真假守则",而是**"禁忌知识在探索中渐揭"**——玩家通过行动发现"有些事不该知道、有些东西不该看"。`discovered` 机制复用(发现一条禁忌知识 → 点亮 + 可能涨 knowledge/解锁力量但掉 san)。规则读起来是"代价",不是"攻略"。
- **结局**:复用 endings 机制——可设计 san 触底=疯狂结局、knowledge 极高=窥见真相的某种代价结局、hp 触底=死亡、以及某种"全身而退/无知是福"的生存结局。AI 提议 + 引擎 §4.4 接受 + §5 兜底(复用零改)。
- **lockstep**:`.md` 资产 + 运行时 Java 副本同步改(沿用 Phase 1/2 教训)。保留叙事清洁度硬约束(禁内部字段名 `knowledge`/`san`/markdown 头进散文)+ 长度约束。

## 4. 卡片专属氛围(顺手做,backlog 流水线那步)

克苏鲁解锁后选择屏自动出第三张卡(registry 驱动,前端零改);配专属氛围:
- **幽绿 / 深渊蓝 + 缓慢扭曲、触手般蠕动的暗影 + 不可名状的不安**。
- 跟规则怪谈"冷蓝诡异"区分:克苏鲁更"深海/古old/疯狂",规则怪谈更"日常诡异"。
- 纯 CSS(不引 GSAP),沿用 A 计划"每 archetype 一套 `.cardXxx` 主题块 + `vibeClass()` 分支"的可扩展形态;守底线(低频、不晃眼、文字可读、`prefers-reduced-motion` 关运动)。

## 5. 切分顺序 + 测试 + 冒烟门

**切分(由纯到脏,极轻)**:
1. registry 加克苏鲁条目(含 knowledge 轴)+ 元数据字段。
2. world-gen + event-loop 注入块(克苏鲁世界观 + knowledge 轴 + 联动提示,lockstep 运行时副本)。
3. 克苏鲁卡片 CSS 氛围。
4. 真 key 冒烟(验收门,§5 冒烟,**待 Felix 在场**)。

**测试(确定性,不打真实 API)**:
- **回归护城河(第一不变量)**:加克苏鲁后 server 156 + frontend 27 全绿——加世界绝不能弄坏规则怪谈/末日。
- 克苏鲁 registry 条目结构 + knowledge 轴 key 集合;选择屏 `listForSelection()` 现在含克苏鲁(active)。
- 引擎无知再确认:`Engine.apply` 对 `{hp,san,knowledge}` 与 `{hp,san}`/`{hp,hunger}` 行为一致(同一通用结算)——证流水线没逼引擎懂新轴。
- 前端面板喂克苏鲁 attributes → 渲染"体力/理智/禁忌知识"中文名(动态渲染已就位,确认三轴也对)。

**冒烟门(真 key · ADR-008 决策 5 · 此关不过不算完成 · 待 Felix 在场)**:
- 跑几个克苏鲁 `init` → world-gen 首次有效率(对照末日/规则怪谈,**不外推**)、格式完整、**knowledge 轴在 attributes 齐**、消毒无泄露。
- 驱回合,**两条专门观测**:
  1. **knowledge 累积**:玩家求知/探索时 AI 是否稳定让 knowledge 上涨?
  2. **联动(本批最关键、最没验过的)**:knowledge 高时,AI 是否真的让 san 流失更快?这是比末日单纯衰减更复杂的"联动型语义",AI 能否稳定执行是真没验过的——若不稳,是影响所有"轴联动型模式"的发现,克苏鲁第一次便宜暴露。
- 克苏鲁叙事泄露人眼扫(禁忌知识题材,首次真实测)。
- 存克苏鲁真实 raw + SSE 转录(untracked)。
- **Felix 读数 + 拍板**:克苏鲁是否验通 / 联动稳不稳 / 体感(禁忌知识的张力出来没)/ 要否调提示词。
