# 更新 ROADMAP

为 Felix 的「AI 后端架构师一年计划」更新 `docs/ROADMAP.md`。这是一份"中央档案",更新要稳、要对齐既有结构,**只增改相关项,不重排版面、不改既有措辞**。

## 最近提交(帮你回忆本次做了什么)

```!
git log --oneline -15 2>/dev/null || echo "非 git 仓库或不在根目录"
```

## 流程

1. **先读 `docs/ROADMAP.md`**,定位要动的几处,其余一律不碰。
2. 确认本次更新的范围(默认从 `$ARGUMENTS` + 上面最近提交推断):某天收尾 / 某周收尾 / 某里程碑达成 / 新 ADR 落地?缺信息就一次性问清,不要逐条挤牙膏。
3. 按下面「更新点清单」逐项改。
4. 改完把 diff 摘要给 Felix 看,点明动了哪几节。

## 更新点清单(只动相关项)

1. **一、项目元信息表**:若阶段 / 月份推进了,更新「当前阶段」「当前月份」两行。

2. **三、进度追踪表(主表)**:更新对应月份行的「状态」、「实际完成日期」(完成时填)、「主要产出 / 链接」、「备注」。状态符号严格用图例四个:⬜ 未开始 / 🟨 进行中 / ✅ 已完成 / ⚠️ 阻塞。

3. **三、周度学习日志**:
   - 给当前周追加或补全条目,沿用既有结构:`**Week N(日期范围)**` + `完成` / `决策` / `卡点`(注明是否已 unblock)/ `下周计划`。
   - 遵守「最近三周,循环覆盖」:近周条目超过三条时,**不要直接删旧的**——提示 Felix 把最旧一周移到 `docs/retrospectives/week-N-retrospective.md`,正文留指针链接。
   - 完成项里凡涉及决策 / 踩坑,带上 `ADR-NNN` / `笔记 N` 的交叉引用,与现有写法一致。

4. **四、ADR 索引**(若本次有新 ADR):在「已完成 ADR 索引」表追加一行;若某条「待决策候选议题」已被该 ADR 解决,从候选列表移除。

5. **九、版本历史**:追加一行 `| v?.? | YYYY-MM-DD | <一句话修订内容> |`。版本号在上一条基础上 +0.1;修订内容对齐既有 v1.x 的粒度——点出本次动了哪些节,而不是流水账。

## 约定

- 日期用 `YYYY-MM-DD`,默认今天。
- 口吻与缩写(M1 / Week / Day、ADR-NNN、笔记 N)对齐文档现有风格。
- **只更新,不重构**:不调章节顺序、不改既有措辞,除非 Felix 明确要求修订。
- **与 `adr-author` 配合**:若本次更新源于新 ADR,提醒 Felix 确认 ADR 正文要求的 `README.md` 列表行和 LEARNING-NOTES 双向链接是否也已同步(那是 `adr-author` 的收尾项,不在本技能职责内)。
